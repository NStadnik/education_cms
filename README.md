# Education CMS

Самостійна PHP CMS для сайтів закладів освіти. Система зроблена без
фреймворків: чистий PHP 8.2+, PDO, серверні шаблони, адмінка, ролі,
новини, сторінки та базові налаштування сайту.

## Локальний запуск

```bash
php -S 127.0.0.1:8080 -t public
```

Відкрийте `http://127.0.0.1:8080/install` і введіть дані MySQL/MariaDB.

Якщо хочете перевірити режим “усе в корені”, запустіть:

```bash
php -S 127.0.0.1:8080
```

У цьому режимі працює кореневий `index.php`, а системні папки закриті через
`.htaccess` на Apache-хостингу.

## Встановлення на хостинг

1. Завантажте файли проєкту на хостинг.
2. Відкрийте домен у браузері або одразу перейдіть на `/install`.
3. Введіть дані підключення до MySQL/MariaDB, назву закладу, email і пароль
   першого адміністратора.
4. Після встановлення приберіть право запису для `config/local.php`, якщо це
   дозволяє панель керування хостингом.

Рекомендований простий варіант для більшості хостингів: завантажити всі файли
в корінь сайту, де вже лежить `index.php`.

Більш ізольований варіант для VPS або гнучкого хостингу: вказати web root на
папку `public/`. Обидва режими підтримуються.

CMS використовує MySQL/MariaDB. На сервері має бути увімкнено PHP-розширення
`pdo_mysql`.

Директорії, які мають бути доступні для запису:

- `storage/`
- `storage/sessions/`
- `storage/uploads/`
- `storage/cache/`

## Основні модулі

- Сторінки з блоковим наповненням.
- Новини.
- Користувачі, ролі та права доступу.
- Журнал дій.
- Інсталятор.

## Інтерфейс

Адмінка та базові екрани використовують Bootstrap 5 через CDN. Локальний файл
`public/assets/site.css` лишається для кольорів, сайдбару та невеликих
проєктних доповнень.

## Шаблони сайту

Глобальні шаблони публічного сайту винесені в окремі папки:

```text
templates/site-themes/{key}/theme.php
public/assets/site-themes/{key}/theme.css
```

`theme.php` описує назву, опис, порядок, можливості та кольори превʼю. CSS-файл
підключається тільки для вибраного шаблону. Щоб додати новий шаблон, створіть
нову пару папок із тим самим `{key}` і виберіть його в адмінці на сторінці
`/admin/templates`.

## Debug-режим

Debug потрібен тільки для відлагодження на хостингу. Не залишайте його
увімкненим на робочому сайті.

Увімкнути debug можна одним із двох способів:

1. Створити файл:

```text
storage/debug.enabled
```

2. Або в `config/local.php` встановити:

```php
'app' => [
    'debug' => true,
],
```

Після цього відкрийте:

```text
/debug
```

На сторінці буде видно версію PHP, SAPI, шляхи, стан сесій, доступність
`storage`, розширення PDO та інші дані середовища.

Лог помилок пишеться сюди:

```text
storage/debug.log
```

Після завершення відлагодження видаліть:

```text
storage/debug.enabled
```

або поверніть:

```php
'debug' => false
```

## Оновлення через GitHub Releases

CMS підтримує перевірку й встановлення оновлень з GitHub Releases в адмінці:

```text
/admin/updates
```

Оновлення доступні користувачам із правами супер адміністратора. Механізм
завантажує zip-архів релізу, перевіряє `.sha256`, створює backup у
`storage/updates/backups/` і замінює лише системні файли.

Під час оновлення не перезаписуються:

- `config/local.php`
- `storage/`
- завантажені файли, сесії, кеш і backup-и

Для автоматичного оновлення на сервері потрібне PHP-розширення `zip`. Для
завантаження з GitHub бажано мати `curl`; якщо його немає, CMS спробує
використати `file_get_contents` із HTTPS.

Щоб створити новий реліз, оновіть версію в `VERSION` і `config/app.php`, після
чого створіть tag:

```bash
git tag v1.0.1
git push origin v1.0.1
```

GitHub Actions збере й прикріпить до релізу:

```text
education-cms-v1.0.1.zip
education-cms-v1.0.1.sha256
```

## Примітки

Адмінка навмисно компактна. Це основа для коробкового продукту під сайти
закладів освіти, а не спроба відтворити повноцінний фреймворк.

## Інтеграція з ЛКЛАУД

Education CMS підтримує одноразовий SSO-вхід викладача з ЛКЛАУД у редактор
новин. Підключення налаштовується в `/admin/settings` у блоці
«Інтеграція з ЛКЛАУД». Вкажіть issuer, audience, URL ЛКЛАУД, окремий
SSO-секрет щонайменше з 32 символів та API-ключ. Значення зі старого
`config/local.php`, якщо вони були задані, автоматично переносяться в БД.

ЛКЛАУД має передати HS256 JWT у `GET /auth/lcloud?token=...`. Обов'язкові
claims: `iss`, `aud`, `sub`, `iat`, `exp`, `jti`; додаткові: `name`, `email`,
`institution_id`. Токен може діяти не більше п'яти хвилин і використовується
лише один раз. Новому користувачу автоматично призначається роль `teacher`,
яка дозволяє працювати тільки з власними новинами та медіафайлами.

Список публікацій доступний через `GET /api/lcloud/publications`. Якщо задано
`api_key`, запит повинен містити `Authorization: Bearer <key>` або
`X-Lcloud-Key: <key>`. Параметр `author_id` фільтрує публікації за `sub`
викладача ЛКЛАУД.
